## How to Contribute

### Reporting Bugs üêõ

1. **Check existing issues** - Don't duplicate
2. **Create new issue** with:
   - Clear title
   - Steps to reproduce
   - Expected vs actual behavior
   - Browser/OS info
   - Screenshots (if visual)
   - Console errors (if applicable)

### Suggesting Features üí°

1. **Check feature list** in FEATURES.md
2. **Open discussion** issue
3. **Describe**:
   - What problem it solves
   - How it should work
   - Impact on gameplay
   - Implementation ideas

### Submitting Code üìù

1. **Fork** the repository
2. **Create branch**: `git checkout -b feature/your-feature`
3. **Make changes** following code style
4. **Test locally** - use DEVELOPMENT.md
5. **Commit** with clear messages
6. **Push** to your fork
7. **Create Pull Request** with:
   - Clear title
   - Description of changes
   - Related issue (if any)
   - Testing notes

### Code Style

Follow existing patterns:
- ES6+ syntax
- camelCase for variables/functions
- PascalCase for classes
- Meaningful names
- Comments explain 'why', not 'what'
- Emoji logging prefixes

### Testing

Before submitting PR:
- [ ] No console errors
- [ ] All levels play correctly
- [ ] No performance degradation
- [ ] Mobile touch works
- [ ] Responsive on all sizes
- [ ] Documentation updated

### Documentation

- Update README if user-facing
- Update DEVELOPMENT.md if technical
- Update CHANGELOG.md with changes
- Add code comments if complex

1. **Ensure tests pass**
2. **Update documentation**
3. **Add to CHANGELOG.md**
4. **Request review**
5. **Address feedback**
6. **Merge when approved**
